gfind.exe -- Find Non‑redirecting Public Links

Show Usage Information

Usage: gfind.exe <start> <end> [--delay N] [--block N] [--title]

--delay # or --d #: delays each link test to ensure whether it redirects or not
--block # or --b #: processes links in blocks of N links, 50 is a good number to prevent errors
--title   or --t  : displays the title of the site alongside the url

Purpose: To find public working sites easier on the internet

> About gfind

Written in C using libcurl, gfind checks ranges of sites for non‑redirecting, public‑facing pages. It performs concurrent link testing, extracts site titles, and can filter out “404”, error pages, etc.

The C version provides much greater speed and efficiency compared to the Python version via using concurrent functions.

> Python Equivalent

A slower, educational Python variant exists: View gfind.py on GitHub

It demonstrates the same logic using requests but lacks multithreading and thus runs more slowly than the C program.

> JSON Formatter — gfind_fmtr.exe

After running gfind and saving your output, you can convert results to JSON using: 

./gfind_fmtr

Ensure there is an input.txt file in the same directory — this should be a copy‑paste of the raw console output from gfind using your chosen parameters. The formatter will read this and produce output.json in the format: 

{
  "data": [
    { "title": "Example Site", "url": "https://sites.google.com/view/100" },
    ...
  ]
}

The json can then be entered into ffdataformater.html by pasting the output.json contents into const jsonData = [output.json contents];. Ctrl-f "const jsonData" on ffdataformater.html and you'll find it. An example ffdataformater.html (using output from ./gfind 0 500 --b 50 --t)
