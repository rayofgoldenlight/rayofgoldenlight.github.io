FHIR Patient Finder

A .NET web API that searches and manages patients following FHIR standards

What This Project Does

The FHIR Patient Finder is a .NET 6+ web application created to search, favorite, and manage patient data following the FHIR (Fast Healthcare Interoperability Resources) standards. It communicates with FHIR-compatible servers to perform patient lookups by name, and provides endpoints to manage favorite patients and audit interactions.

Core Technologies Used

  • ASP.NET Core Minimal API
  • Entity Framework Core (SQL Server LocalDB)
  • Custom FHIR client service
  • Modern HTML/CSS/Bootstrap frontend

Database Used

The backend uses an older local version of Microsoft SQL Server (LocalDB). You can find info or install it from the official Microsoft link: https://aka.ms/sqllocaldb.

Key Features

  • Search patients by name (via FHIR API)
  • Add and remove patient favorites
  • Track actions in an audit log
  • Simple RESTful API structure for easy integration

Implementation Highlights

The project uses dependency injection for all core services:

  • IFhirPatientService – handles FHIR API requests
  • IPatientRepository – handles database and favorite management
  • AppDbContext – Entity Framework context for audit logging and patient records

Each endpoint is built as a minimal API route, for simplicity and clarity. The frontend (in a separate project) uses these endpoints to perform real-time searches and favorite operations.

How to Build and Run the Project

For users who’d like to explore and run the FHIR Patient Finder source code themselves, you can follow these simple steps below. Click to expand for details.

  1. Make sure you have the latest version of the .NET SDK installed.
  2. Clone the project from GitHub: 
    git clone https://github.com/rayofgoldenlight/tree.git
  3. Navigate to the project folder: 
    cd projects/fhirpatientsearch/fpf
  4. Build the project: 
    dotnet build
  5. Run the application: 
    dotnet run
  6. Open your browser and visit the shown local URL (usually https://localhost:5001) to access the API or web UI.

💡 You may need to ensure your local database connection is configured correctly. This project uses SQL LocalDB.
